Команды Git
Команда Описание
git init Создает новый репозиторий Git
git clone url Клонирует репозиторий
git add file Cохраняет снимок текущего состояния в проекта в истории коммитов
git add . Индексация изменения
git commit -m "comment" Комментируем коммит прямо из командной строки вместо текстового редактора -m : сообщение
git commit -a -m "comment" –a означает: добавить все изменения в индекс до передачи
git push url Позволяет отправлять локальную ветку на удаленный репозиторий
git status Отображает состояние рабочего каталога и раздела проиндексированных файлов
git log git log — это путь. Указав имя каталога или файла, вы ограничите вывод log теми коммитами, которые вносят изменения в указанные файлы
git branch branch_name Используется для создания новой ветки
git checkout branch_name Переключение между различными версиями целевого объекта
git checkout -b branch_name Создает ветку от текущего указателя
git remote Используется для работы с внешними репозиториями

Шаблоны gitignore
Шаблон Пояснение
**/logs Добавьте в начало шаблона две звездочки, чтобы сопоставлять каталоги в любом месте репозитория.
**/logs/debug.log Две звездочки можно также использовать для сопоставления файлов на основе их имени и имени родительского каталога.
*.log Одна звездочка — это подстановочный знак, который может соответствовать как нескольким символам, так и ни одному.
debug.log По умолчанию шаблоны соответствуют файлам, находящимся в любом каталоге
logs Без косой черты в конце этот шаблон будет соответствовать и файлам, и содержимому каталогов с таким именем. В примере соответствия слева игнорируются и каталоги, и файлы с именем logs
logs/**/debug.log Две звездочки соответствуют множеству каталогов или ни одному.
logs/*day/debug.log Подстановочные символы можно использовать и в именах каталогов.
